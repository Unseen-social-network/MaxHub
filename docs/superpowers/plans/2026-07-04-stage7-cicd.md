# Stage 7 — CI/CD Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Automate verification on every push/PR and automate GHCR build + server deploy strictly on version tags.

**Architecture:** `.github/workflows/ci.yml` runs on every `push`/`pull_request`: a `test` job (Postgres service container, `uv sync`, `ruff check`, `ruff format --check`, `pytest` — migrations apply automatically via the existing session-scoped `tests/db/conftest.py` fixture, no separate migration step needed) and a parallel `docker-build` job that builds the Stage 6 `Dockerfile` without pushing, both gating merges. `.github/workflows/deploy.yml` triggers **only** on `push: tags: ['v*']`: builds and pushes the image to GHCR under `{tag}` and `latest` (image name matches `deploy/docker-compose.yml`'s hardcoded `ghcr.io/unseen-social-network/maxhub`, derived from the real `origin` remote), then SSHes into the server and runs the exact `sed`/`docker compose pull && up -d && image prune` sequence from CLAUDE.md.

**Tech Stack:** GitHub Actions — `actions/checkout@v7`, `astral-sh/setup-uv@v8`, `docker/setup-buildx-action@v3`, `docker/build-push-action@v7`, `docker/login-action@v4`, `appleboy/ssh-action@v1`. (Action major versions confirmed current via each action's README as of 2026-07 — this environment's package/action ecosystem is dated 2026, meaningfully newer than commonly-cached training knowledge, so versions were verified rather than assumed.)

## Global Constraints

- `ci.yml`: on push and PR — `uv sync`, `ruff check`, `ruff format --check`, `pytest` (Postgres as a service container), `docker build` without push (CLAUDE.md "Этап 7" / "Версионирование и деплой")
- `deploy.yml`: trigger strictly `on: push: tags: ['v*']` — no deploys from branches (CLAUDE.md "Этап 7" / "Версионирование и деплой")
- Deploy steps: build with build-args (`APP_VERSION` = `${{ github.ref_name }}`, `GIT_SHA`, `BUILD_TIME`) → push to GHCR tagged `{tag}` and `latest` → SSH via `appleboy/ssh-action` (secrets `SSH_HOST`, `SSH_USER`, `SSH_KEY`, optional `SSH_PORT`) → on the server: `cd /opt/maxbot && sed -i "s/^IMAGE_TAG=.*/IMAGE_TAG={tag}/" .env && docker compose pull && docker compose up -d && docker image prune -f` (CLAUDE.md "Этап 7", exact commands)
- Commit messages: Russian, Conventional Commits, no AI-authorship trailers (CLAUDE.md Section 0)

### Task 1: `ci.yml`

**Files:**
- Create: `.github/workflows/ci.yml`

**Interfaces:**
- Produces: two parallel jobs, `test` and `docker-build`, both triggered on `push` and `pull_request`.

- [ ] **Step 1: Write `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
  pull_request:

jobs:
  test:
    runs-on: ubuntu-latest

    services:
      postgres:
        image: postgres:16
        env:
          POSTGRES_USER: maxhub
          POSTGRES_PASSWORD: maxhub
          POSTGRES_DB: maxhub
        ports:
          - 5432:5432
        options: >-
          --health-cmd pg_isready
          --health-interval 10s
          --health-timeout 5s
          --health-retries 5

    env:
      BOT_TOKEN: test-token
      DOMAIN: example.com
      WEBHOOK_PATH: /webhook
      PORT: "8080"
      MODE: polling
      ADMIN_IDS: "1"
      DATABASE_URL: postgresql+asyncpg://maxhub:maxhub@localhost:5432/maxhub

    steps:
      - uses: actions/checkout@v7

      - name: Install uv
        uses: astral-sh/setup-uv@v8
        with:
          python-version: "3.12"
          enable-cache: true

      - name: Install dependencies
        run: uv sync --frozen

      - name: Lint
        run: uv run ruff check .

      - name: Format check
        run: uv run ruff format --check .

      - name: Run tests
        run: uv run pytest -v

  docker-build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v7

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build image
        uses: docker/build-push-action@v7
        with:
          context: .
          push: false
          build-args: |
            APP_VERSION=ci
            GIT_SHA=${{ github.sha }}
            BUILD_TIME=ci
```

Expected: `OK`

### Task 2: `deploy.yml`

**Files:**
- Create: `.github/workflows/deploy.yml`

**Interfaces:**
- Produces: one `deploy` job, triggered only on tags matching `v*`.

- [ ] **Step 1: Write `.github/workflows/deploy.yml`**

```yaml
name: Deploy

on:
  push:
    tags:
      - "v*"

permissions:
  contents: read
  packages: write

env:
  IMAGE_NAME: ghcr.io/unseen-social-network/maxhub

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v7

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Log in to GHCR
        uses: docker/login-action@v4
        with:
          registry: ghcr.io
          username: ${{ github.actor }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Build timestamp
        id: build_time
        run: echo "value=$(date -u +%Y-%m-%dT%H:%M:%SZ)" >> "$GITHUB_OUTPUT"

      - name: Build and push
        uses: docker/build-push-action@v7
        with:
          context: .
          push: true
          tags: |
            ${{ env.IMAGE_NAME }}:${{ github.ref_name }}
            ${{ env.IMAGE_NAME }}:latest
          build-args: |
            APP_VERSION=${{ github.ref_name }}
            GIT_SHA=${{ github.sha }}
            BUILD_TIME=${{ steps.build_time.outputs.value }}

      - name: Deploy over SSH
        uses: appleboy/ssh-action@v1
        with:
          host: ${{ secrets.SSH_HOST }}
          username: ${{ secrets.SSH_USER }}
          key: ${{ secrets.SSH_KEY }}
          port: ${{ secrets.SSH_PORT }}
          script: |
            cd /opt/maxbot
            sed -i "s/^IMAGE_TAG=.*/IMAGE_TAG=${{ github.ref_name }}/" .env
            docker compose pull
            docker compose up -d
            docker image prune -f
```

- [ ] **Step 2: Validate YAML syntax**

Run: `uv run python -c "import yaml; yaml.safe_load(open('.github/workflows/deploy.yml'))" && echo OK`
Expected: `OK`
